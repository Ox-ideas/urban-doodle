Schema is the contract that defines how the LLM Wiki should be structured, maintained, and used.

## Purpose
- enforce consistent folder structure and naming
- define required metadata and frontmatter
- specify how sources are ingested and wiki pages are authored
- guide agent behavior and vault automation

## Components
- folder conventions for `raw/`, `wiki/`, and `wiki/*`
- page type conventions such as `concept`, `entity`, `source-summary`, and `comparison`
- frontmatter field requirements
- workflow rules for ingest, query, and maintenance
