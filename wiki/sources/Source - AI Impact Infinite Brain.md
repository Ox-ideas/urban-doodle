# Source: Don't Use Karpathy's Second Brain (I BUILT SOMETHING BETTER)

A YouTube transcript by [[AI Impact]] discussing the evolution from the "Second Brain" (PARA/PKM) method to a more AI-native, knowledge-graph-based approach for personal and business knowledge management.

## Key Takeaways
- **Knowledge Graphs for AI**: Emphasizes that high-quality AI requires a structured, linked knowledge base tailored to the user or business.
- **Obsidian as a Tool**: Demonstrates using Obsidian to visualize and manage knowledge graphs.
- **From PARA to Infinite Brain**: Critiques the PARA (Projects, Areas, Resources, Archives) method as human-centric and introduces the "Infinite Brain" system, which is more granular and AI-friendly.
- **Atomic Notes & Types**: Advocates for small, atomic notes (50–300 tokens) and introduces 16 node types (pillars, decisions, concepts, questions, playbooks, etc.) for better AI comprehension.
- **Complex Edge Types**: Proposes richer link types (supports, contradicts, depends on, derived from, part of, authored, etc.) to clarify relationships between notes for AI.
- **Token Efficiency**: Shows that the Infinite Brain structure is far more token-efficient for AI retrieval and reasoning than traditional PKM.
- **Practical Use Cases**: Describes using this system for SOPs, design, analysis, and as a persistent, evolving knowledge base for both humans and AI.
