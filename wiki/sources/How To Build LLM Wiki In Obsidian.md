## Summary
This source is a transcript of a workflow-oriented video showing how to set up an LLM Wiki in Obsidian. It documents the three core layers (raw sources, wiki, schema), the folder structure, templates, tooling, ingest process, maintenance loop, and advanced agentic features.

## Key points
- Build a new vault and initialize it with git for safe version control.
- Create the folder structure for raw sources, wiki pages, and schema files.
- Define the schema, templates, and agent instructions that enforce consistent note creation.
- Use scripts and skills to automate ingestion, cataloging, maintenance, and linting.
- Perform a test ingest on a raw source, then run maintain/lint/query to verify the system.
- Advanced features include vault-safe wrappers, local model workflows, and review slabs for human-in-the-loop approval.
